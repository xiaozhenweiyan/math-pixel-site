# 新增负数支持方法 + 过拟合 + 偏移算法 Spec

## Why
用户报告负数序列预测效果差，需要新增 10 种百分百支持负数的预测方法；同时希望数据不足时显示还差几个数据；并新增两个独立算法面板（过拟合算法、偏移算法），均不计入融合总答案。

## What Changes
- 新增 10 种完全支持负数的预测方法（predictors.js），总数达到 40 种
- 方法详情列表中「数据不足」改为「还差 X 个」，显示需要补充的数据量
- 新增「过拟合算法」面板（折线图下方）：三种过拟合方法（最高次多项式插值、多高次平均、三次样条插值），取最优两个加权平均，单独显示预测答案，不计入融合
- 新增「偏移算法」面板（过拟合下方）：对数据做水平+垂直平移，在 10+ 种简单函数族中找拟合最好的，显示函数表达式和预测值，不计入融合
- 更新标题、方法数量文案为「40 种」

## Impact
- Affected code: `js/predictors.js`（+10 方法）、`js/app.js`（渲染 + 状态）、`index.html`（新增面板 + 标题）、`styles/pixel.css`（新面板样式）
- 新增文件：`js/overfit.js`（过拟合算法）、`js/offsetfit.js`（偏移算法）

## ADDED Requirements

### Requirement: 10 种负数支持预测方法
系统 SHALL 新增 10 种完全支持负数的预测方法，所有方法在全负数、混合正负序列上均能正常计算。

#### Scenario: 全负数序列预测
- **WHEN** 用户输入全负数序列（如 -5, -3, -1, 1, 3）
- **THEN** 40 种方法全部能给出预测结果，无任何方法返回 null / 数据不足

### Requirement: 数据不足显示还差几个
系统 SHALL 在方法详情列表中，对数据不足的方法显示「还差 X 个」而非笼统的「数据不足」。

#### Scenario: 方法缺数据
- **WHEN** 某方法 minLen=5 但当前只有 3 个数据点
- **THEN** 该方法预测值列显示「还差 2 个」

### Requirement: 过拟合算法面板
系统 SHALL 在折线图下方新增「过拟合算法」面板，包含三种过拟合方法，取最优两个加权平均，单独显示答案，不参与融合。

三种过拟合方法：
1. 最高次多项式插值（n-1 次 Lagrange/Newton 插值，精确通过所有点）
2. 高次多项式平均（deg=5,6,7,8 分别预测取平均）
3. 三次样条插值（Cubic Spline，自然边界条件）

集成策略：按留一误差反权重，取全部三个方法的加权平均。

#### Scenario: 过拟合预测
- **WHEN** 用户输入数据并点击预测
- **THEN** 过拟合面板显示三个子方法的结果 + 加权融合结果
- **AND** 该结果不计入上方「融合预测结果」

### Requirement: 偏移算法面板
系统 SHALL 在过拟合算法下方新增「偏移算法」面板，将数据视为整体函数 f(x)，通过水平位移 h + 垂直位移 k 使 y[i] ≈ g(i - h) + k，其中 g 为简单函数族中拟合最好的一个。

简单函数族（10+ 种）：
1. 常数：g(x) = c
2. 线性：g(x) = a·x + b
3. 二次：g(x) = a·x² + b·x + c
4. 三次：g(x) = a·x³ + b·x² + c·x + d
5. 绝对值：g(x) = a·|x - b| + c
6. 倒数：g(x) = a / (x - b) + c
7. 指数：g(x) = a·exp(b·x) + c（通过符号检测避免负数取对数问题）
8. 幂函数：g(x) = a·x^b + c（x 平移到正半轴后拟合）
9. 正弦：g(x) = a·sin(b·x + c) + d
10. 平方根：g(x) = a·sqrt(|x - b|) + c
11. 双曲正切：g(x) = a·tanh(b·x + c) + d

选择策略：对每个函数族，用网格搜索 + 最小二乘找到最佳 (h, k) 和函数参数，选残差最小的函数族。若残差低于阈值（如 R² > 0.99）则认为「完完整整是一个简单函数」，否则显示「最接近」。

#### Scenario: 偏移算法预测
- **WHEN** 用户输入数据并点击预测
- **THEN** 偏移算法面板显示：找到的简单函数表达式（含平移参数）、拟合优度、预测值
- **AND** 该结果不计入上方「融合预测结果」
