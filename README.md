# 像素预测器 PIXEL PREDICTOR

> 一个采用「复古深空像素风（Retro Deep-Space Pixel）」的纯静态网页应用：输入一个数字序列，使用 20 种数学方法预测下一个数字，并展示各方法权重与折线图。可直接部署到 GitHub Pages，无需构建、无需后端。

## 特性

- **20 种数学预测方法**：朴素法、漂移法、移动平均、指数平滑（Holt / Holt-Winters）、多项式回归、自回归 AR(1)/AR(2)、几何增长、差分外推、Fibonacci 黄金比率、傅里叶外推等
- **动态权重计算**：基于留一法回测（Leave-One-Out Backtesting）的 MAPE 误差反推每种方法的权重
- **双权重模式**：支持「回测权重」与「均匀权重」实时切换对比
- **像素风可视化**：原生 Canvas 绘制折线图与权重条形图，无第三方图表库
- **数据导出**：支持导出为 JSON / CSV
- **响应式布局**：桌面双栏 / 移动端单栏自适应
- **复古深空像素风 UI**：深紫黑底 + 白色粗边 + 硬阴影无模糊 + 等宽字体 + 微圆角

## 20 种预测方法

| # | 方法 | 类别 | 原理 |
|---|------|------|------|
| 1 | 朴素法 Naive | 基础 | 取序列最后一个值作为预测 |
| 2 | 季节朴素法 Seasonal Naive（周期=2） | 基础 | 取上上个值（假设周期为 2 的季节性） |
| 3 | 漂移法 Drift | 基础 | 用首末两点斜率外推一步 |
| 4 | 简单平均 Mean | 基础 | 所有值的算术平均 |
| 5 | 中位数 Median | 基础 | 所有值的中位数 |
| 6 | 简单移动平均 SMA（window=3） | 平滑 | 最近 3 个值的平均 |
| 7 | 加权移动平均 WMA（window=3） | 平滑 | 最近 3 个值按 1:2:3 加权平均 |
| 8 | 简单指数平滑 SES（α=0.5） | 平滑 | 递推 `s = α·x + (1-α)·s` |
| 9 | 二次指数平滑 Holt's Linear | 平滑 | 同时平滑水平与趋势（α=β=0.3） |
| 10 | 三次指数平滑 Holt-Winters（周期=2） | 平滑 | 同时平滑水平、趋势、季节性（α=β=γ=0.3） |
| 11 | 线性回归 Linear Regression | 回归 | 最小二乘拟合直线外推 |
| 12 | 二次多项式回归 Poly2 | 回归 | 拟合抛物线外推（minLen=3） |
| 13 | 三次多项式回归 Poly3 | 回归 | 拟合三次曲线外推（minLen=4） |
| 14 | 自回归 AR(1) | 自回归 | 用前一个值预测下一个（minLen=3） |
| 15 | 自回归 AR(2) | 自回归 | 用前两个值预测下一个（minLen=4） |
| 16 | 几何增长 Geometric | 其他 | 平均比率法，最后一个值 × 平均比率 |
| 17 | 一阶差分外推 Diff1 | 其他 | 用最后一阶差分外推 |
| 18 | 二阶差分外推 Diff2 | 其他 | 用二阶差分（差分的差分）外推 |
| 19 | Fibonacci / 黄金比率 | 其他 | 检测比率是否接近 φ=1.618，是则用 φ 外推 |
| 20 | 傅里叶外推 Fourier（k=3） | 其他 | DFT 取前 3 个谐波重建并外推（minLen=6） |

## 权重计算

系统使用**留一法回测（Leave-One-Out Backtesting）**评估每种方法在历史数据上的表现：

1. 对每种方法，依次用前 i 个数据点预测第 i+1 个点（i 从 minLen 到 n-1）
2. 计算所有预测点的 **MAPE**（平均绝对百分比误差）：
   ```
   MAPE = (1/m) × Σ |actual - predicted| / |actual|
   ```
3. 权重 = `1 / (MAPE + ε)`，归一化使所有方法权重之和为 1（ε=1e-10 防止除零）
4. 数据不足或回测失败的方法权重为 0

**融合预测** = Σ (方法预测值 × 方法权重)

支持切换为「均匀权重」模式：所有有效方法平均权重，便于对比回测权重的效果。

## GitHub Pages 部署

1. Fork 或 clone 本仓库到你的 GitHub 账号
2. 推送到 `main` 分支
3. 进入仓库 **Settings → Pages**
4. **Source** 选择 `Deploy from a branch`
5. **Branch** 选择 `main` / `(root)`
6. 点击 **Save**
7. 等待 1-2 分钟，访问 `https://<你的用户名>.github.io/<仓库名>/`

无需任何额外配置，纯静态文件直接托管。

## 技术栈

- 纯 HTML / CSS / JavaScript（ES5+ 兼容）
- 原生 Canvas API（**无图表库**，不引入 Chart.js / D3 / p5.js）
- 无构建步骤、无后端依赖、无第三方运行时库
- 可直接部署到 GitHub Pages

## 安全

- 所有用户输入通过 `textContent` / `createElement` 渲染，不使用 `innerHTML` 拼接（防 XSS）
- 输入数值边界处理（极大、极小、负数、小数、零）
- 所有计算结果在显示前过滤 `NaN` / `Infinity`
